# Конфигурация

`config.yaml` описывает три верхнеуровневых секции, которые ссылаются друг на
друга по строковым id:

```yaml
lora: [...]          # массив физических LoRa-нод
messengers: [...]    # массив мессенджер-транспортов
rooms: [...]         # массив логических комнат
```

## Как это устроено

* В `lora` объявляете физические радиоузлы и их каналы (эндпоинты).
* В `messengers` объявляете ботов мессенджеров (пока только Telegram).
* В `rooms` связываете один LoRa-эндпоинт с подписчиками — чатами мессенджеров
  или другими LoRa-эндпоинтами (LoRa↔LoRa relay).

Каждая комната — это один канал общения. Внутри комнаты сообщения зеркалятся
между всеми участниками: написал в Telegram → ушло в эфир LoRa и в остальные
Telegram-чаты-подписчики; принято из LoRa → разошлось всем подписчикам.

## Минимальный пример

```yaml
lora:
  - id: mc-1
    type: meshcore
    connection:
      type: usb
      device_id: "0403:6015"
    endpoints:
      general:
        type: public
        channel_name: "General"
    policies:
      egress_rate:
        msgs_per_window: 6
        window_seconds: 60

messengers:
  - id: tg
    kind: telegram
    token: ${TG_BOT_TOKEN}

rooms:
  - lora:
      node: mc-1
      endpoint: general
    subscribers:
      - transport: tg
        chat: "-1001234567890"
```

Этот конфиг описывает одну MeshCore-ноду по USB с одним публичным каналом
`General`, одного Telegram-бота, и одну комнату, которая соединяет канал
`general` ноды `mc-1` с Telegram-чатом `-1001234567890`.

## Допустимые формы комнаты

Жёсткий инвариант: либо «1 LoRa + N мессенджеров», либо «2 LoRa + 0 мессенджеров».
Смешанные формы запрещены.

| Форма                    | Когда использовать                                       |
|--------------------------|----------------------------------------------------------|
| 1 LoRa + N мессенджеров  | Типовой мост: один радиоканал ↔ один или несколько чатов |
| 2 LoRa, 0 мессенджеров   | Рилей между двумя радиосетями (LoRa↔LoRa)                |

Пример LoRa↔LoRa:

```yaml
rooms:
  - lora:
      node: meshcore-1
      endpoint: general
    subscribers:
      - lora:
          node: meshcore-2
          endpoint: relay
```

## Подстановка переменных окружения

В YAML работает синтаксис `${ENV_VAR}` через `envyaml`. Секреты держите в
переменных окружения, не в файле:

```yaml
messengers:
  - id: tg
    kind: telegram
    token: ${TG_BOT_TOKEN}        # значение берётся из окружения
```

## Что дальше — справочник полей

Подробное описание каждой секции (и всех полей внутри неё) генерируется
автоматически из pydantic-схемы. Любое описание, которое вы здесь видите, лежит
в коде в виде `Field(description=...)` — это исключает рассинхрон документации
с реальной валидацией.

| Секция        | Что описывает                                | Подробнее                        |
|---------------|----------------------------------------------|----------------------------------|
| `lora`        | Физические LoRa-ноды и их каналы             | [Поля lora](lora.md)             |
| `messengers`  | Мессенджер-транспорты                        | [Поля messengers](messengers.md) |
| `rooms`       | Привязка каналов к подписчикам               | [Поля rooms](rooms.md)           |
| (типы-ссылки) | Идентификаторы, которыми секции ссылаются    | [Типы-ссылки](types.md)          |
