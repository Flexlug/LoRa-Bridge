# LoRa-Bridge

**Двунаправленный мост между LoRa-сетями и мессенджерами.** Сообщения, отправленные
в Telegram-чат, уходят в эфир по LoRa-радиоканалу, а принятое из эфира зеркалится
обратно в Telegram. Несколько физических радиоузлов и каналов поддерживаются
одновременно.

## Что поддерживается

* **LoRa-устройства с прошивкой [MeshCore](https://github.com/ripplebiz/MeshCore)**
  — через USB, serial-порт, TCP-сервер companion или BLE.
* Три типа каналов MeshCore: публичные (`public`), приватные с PSK (`private`),
  Room Server с ACK и backfill (`room_server`).
* **Telegram** — боты через Bot API, групповые чаты, темы (topics), реакции как
  статус доставки.
* Несколько физических нод и каналов одновременно; LoRa↔LoRa мостинг без мессенджера.
* Текстовые сообщения. Вложения (фото/голос) не поддерживаются — вне скоупа.
