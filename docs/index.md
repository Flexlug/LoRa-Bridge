# LoRa-Bridge

**Двунаправленный мост между LoRa-сетями и мессенджерами.** Сообщения, отправленные
в Telegram-чат, уходят в эфир по LoRa-радиоканалу, а принятое из эфира зеркалится
обратно в Telegram. Несколько физических радиоузлов и каналов поддерживаются
одновременно.

## Для чего это нужно

LoRa — низкоскоростной радиоканал дальнего действия, который работает там, где нет
сотовой связи: в лесу, на воде, в горах, между удалёнными станциями. Слабое место —
нет привычного чат-интерфейса, читать с телефона неудобно.

LoRa-Bridge решает это: к одной точке моста подключаются LoRa-устройства, к другой —
Telegram-чат. Группа на местности общается по радио, а наблюдатели в любом месте с
интернетом видят всё в обычном чате (и могут отвечать туда же — ответы дойдут до
радио-устройств в эфире).

Типовые сценарии:

* туристические, охотничьи или поисковые группы — координация в полевых условиях;
* несколько удалённых станций с радиоустройством, объединённые в один чат;
* мост между двумя LoRa-сетями разных регионов/частот.

## Что поддерживается

* **LoRa-устройства с прошивкой [MeshCore](https://github.com/ripplebiz/MeshCore)**
  — через USB, serial-порт, TCP-сервер companion или BLE.
* Три типа каналов MeshCore: публичные, приватные (с PSK), Room Server (с реальным
  ACK и backfill).
* **Telegram** — боты через Bot API, с поддержкой групповых чатов и тем (topics).
* Текстовые сообщения. Вложения (фото/голос) не поддерживаются — это вне скоупа.
