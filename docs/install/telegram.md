# Настройка Telegram-бота

## 1. Создать бота через BotFather

В Telegram найдите официального бота **[@BotFather](https://t.me/BotFather)** и
выполните команду:

```
/newbot
```

Дальше BotFather попросит:

1. Имя бота — то, что увидят пользователи (например, `LoRa Bridge`).
2. Username — должен заканчиваться на `bot`, например `mylora_bridge_bot`.

После этого BotFather пришлёт сообщение с **token** — длинной строкой вида
`123456789:AAA-BBB-...`. Сохраните его — это секрет, который пойдёт в переменную
окружения `TG_BOT_TOKEN`.

## 2. Отключить privacy mode

**Это обязательный шаг для групповых чатов.** По умолчанию боты Telegram не видят
обычные сообщения в группах — только команды, обращённые лично к ним.

В чате с BotFather:

```
/setprivacy
```

Выберите вашего бота из списка и нажмите **Disable**.

!!! warning
    Без этого бот ничего не увидит в группе и LoRa-Bridge не сможет зеркалить
    сообщения. Если уже добавили бота в чат до изменения настройки — удалите его
    и добавьте заново, иначе изменение не применится.

## 3. Добавить бота в нужный чат

Откройте групповой чат → Настройки → Добавить участников → найдите своего бота по
username. Каких-то особых прав боту давать не требуется — обычных прав участника
хватает, чтобы он мог читать и отправлять сообщения.

## 4. Узнать chat_id

LoRa-Bridge должен знать **числовой `chat_id`** чата (не username). Для групп он
выглядит как отрицательное число вида `-1001234567890`.

### Способ 1: показать id прямо в Telegram Desktop

В Telegram Desktop есть скрытая «экспериментальная» панель, которая включает
отображение id у чатов и сообщений.

1. Откройте `Настройки` → `Продвинутые настройки` → `Экспериментальные настройки`
   (в английской версии: `Settings` → `Advanced` → `Experimental settings`).
2. Включите переключатель **`Show Peer IDs in Profile`** (или похожий пункт про
   peer id — название может слегка меняться от версии).
3. Откройте профиль нужного чата — рядом с заголовком/username появится числовой id.

Для супергрупп тот id, который покажет Telegram, — это «короткая» форма, и в
конфиге его нужно записывать с префиксом `-100`. Если в профиле видите
`1234567890`, в `config.yaml` пишите `"-1001234567890"`.

### Способ 2: через @userinfobot

В Telegram добавьте в чат **[@userinfobot](https://t.me/userinfobot)** или
**@JsonDumpBot**, перешлите любое сообщение из своего чата ему лично — он покажет
`chat_id` источника.

### Способ 3: через Bot API

После того как кто-то написал в чат с уже добавленным ботом, откройте в браузере:

```
https://api.telegram.org/bot<ВАШ_TOKEN>/getUpdates
```

В JSON-ответе найдите `chat.id` — это и есть нужное число.

## 5. (Опционально) Темы (topics / threads)

Если в чате включены темы (это «супергруппы с форумом»), для каждой темы есть свой
`topic_id` (число). Получить его можно через **@JsonDumpBot**: переслать ему
сообщение из конкретной темы — в JSON будет поле `message_thread_id`.

В конфиге:

```yaml
rooms:
  - lora:
      node: meshcore-1
      endpoint: general
    subscribers:
      # работа только с этой темой
      - transport: telegram-main
        chat: "-1001234567890"
        topic: "42"
      # вторая подписка — только с General (без topic)
      - transport: telegram-main
        chat: "-1001234567890"
```

* `topic` указан → bridge берёт/шлёт сообщения **только в эту тему**.
* `topic` опущен → bridge работает **только с General** (стандартный канал чата без
  тем).

## 6. Прописать в config.yaml

```yaml
messengers:
  - id: telegram-main      # любое имя; на него ссылаются rooms[].subscribers[].transport
    kind: telegram
    token: ${TG_BOT_TOKEN}
```

`tag` опционально переопределяет короткий тег источника в префиксе LoRa-сообщений
(по умолчанию — заглавные первых двух букв `kind`, то есть `"TG"`):

```yaml
  - id: telegram-main
    kind: telegram
    token: ${TG_BOT_TOKEN}
    tag: "tg"
```

Полная справка по полям мессенджеров — **[здесь](../config/messengers.md)**.
